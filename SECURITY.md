# Security Policy

## Sensitive Data

Do not commit:

- Claude `sessionKey` values.
- Cookies, tokens, API keys or bearer tokens.
- `.env` files.
- Local runtime JSON files such as `settings.json`, `notifications.json` or history files.
- Logs generated by the app.
- Certificates or private keys.
- Published binaries and installer artifacts.

The application stores sensitive provider credentials through Windows Credential Manager. Normal settings and history are stored locally under `%LOCALAPPDATA%\AIUsageWidget`.

## Claude Integration

The Claude integration is non-official and uses isolated web endpoint access. It may break if Claude changes authentication, session handling or response formats.

The application must not log session keys, cookies, authorization headers or full credential values.

## Reporting Security Issues

If this repository is public, report security issues privately to the repository owner instead of opening a public issue with sensitive details.
